# Когортный и RFM анализ на примере датасета покупок аптечной сети
Анализ производился в DBeaver, PostgreSQL

## Когортный анализ LTV
Определение средней ценности клиента по месяцам с момента его первой покупки за период 180 дней.

Временные шкалы каждые 30 дней.

### Этапы исследования:
- Определение "нулевого дня" - дата первой покупки.
- подсчет разницы в днях между первой и текущей покупкой
- дата первой покупки округляется до первого числа месяца для отнесения пользователя к когорте и дальнейшей группировке
- подсчет "чистой" **накопительной** прибыли из расчета количества клиентов в каждой когорте и разбиении по временным интервалам

### Когортный анализ ARPU и ARPPU
продолжаем подсчитывать ценность клиента, теперь уже в разрезе активных клиентов каждой когорты и выручки внутри каждого отрезка в 30 дней

### Этапы исследования:
- Определение "нулевого дня" - дата первой покупки.
- подсчет разницы в днях между первой и текущей покупкой
- дата первой покупки округляется до первого числа месяца для отнесения пользователя к когорте и дальнейшей группировке
- подсчет общего числа клиентов в каждой когорте и активных клиентов внутри каждых 30 дней
- подсчет "чистой" прибыли из расчета количества клиентов в каждой когорте, разбиении по временным интервалам в соотношении к общему числу пользователей в когорте (ARPU) и активных пользователей в каждой когорте (ARPPU).
